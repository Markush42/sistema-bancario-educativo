# Sistema Bancario Educativo — Documentación

Este repositorio contiene la **documentación oficial** de un sistema bancario educativo
diseñado para entender cómo funciona un banco desde cero utilizando la programación.

## Objetivo

- Modelar los conceptos esenciales de un banco:
  - Empleados
  - Clientes
  - Cuentas bancarias
  - Operaciones (depósitos, extracciones, transferencias)
  - Auditoría básica
- Servir como base para un backend real (por ejemplo, con Java/Spring) en futuras etapas.

## Estructura de la documentación

Toda la documentación se encuentra en la carpeta `docs/`:

- `01-vision-alcance.md` — Visión general del sistema y alcance del MVP.
- `02-actores.md` — Actores del sistema (empleados, admin, etc.).
- `03-casos-uso.md` — Casos de uso de alto nivel.
- `04-requisitos-funcionales.md` — Requisitos funcionales (RF-01, RF-02, ...).
- `05-requisitos-no-funcionales.md` — Seguridad, rendimiento, auditoría, etc.
- `06-user-stories.md` — Historias de usuario.
- `07-entidades-dominio.md` — Entidades del dominio bancario.
- `08-entidades-detalle.md` — Atributos y tipos de datos por entidad.
- `09-modelo-conceptual.md` — Descripción del modelo conceptual / UML.
- `10-arquitectura-capas.md` — Arquitectura en capas del sistema.
- `11-modulos-responsabilidades.md` — Módulos lógicos y responsabilidades.
- `12-plan-sprints-testing-despliegue.md` — Plan de sprints, testing y despliegue.
